# 重构3
1. 放弃集成 CloudKit JS，改为直接使用 supabase 存储数据, 宝宝自身数据和成长记录都存储在 supabase 中, 首页页面上可以复制宝宝的ID (UUID), 通过这个ID在其他设备上导入宝宝数据, 这样可以实现跨平台数据同步, 同时也不依赖于 Apple 生态
2. 保留现有的导入,导出功能, 再添加一个直接通过宝宝ID导入的功能, 导入后会新增一个宝宝, 这个宝宝的数据就是其他用户复制的ID对应的宝宝数据, 这样用户就可以通过分享宝宝ID的方式来实现数据迁移和备份
3. 通过ID导入宝宝数据时, 用户应该可以选择是否与原宝宝数据同步, 如果选择同步, 那么以后这个宝宝的数据会和原宝宝数据保持一致, 也就是说在一个设备上修改了这个宝宝的数据, 另一个设备上这个宝宝的数据也会相应修改; 如果选择不同步, 那么导入后这个宝宝的数据就是独立的, 以后在一个设备上修改这个宝宝的数据不会影响另一个设备上的这个宝宝数据
4. 把存储宝宝数据的表结构生成到 README.md 中, 方便用户在自己的 supabase 项目中创建表
5. 首页(我的宝宝页面), 在宝宝卡片下面不要直接展示编辑和删除按钮, 最好可以在卡片左滑出编辑和删除按钮, 这样可以避免误触
6. 删除宝宝的时候不要从数据库中删除, 直接从前端remove这个宝宝就可以, 删除测量数据时可以直接删掉, 用户如果误删宝宝, 可以通过以前保存的ID恢复数据, 相当于通过ID导入宝宝数据
7. 导出的按钮改一下样式, 和导入的按钮相反, 是向上的箭头, 这样更符合用户的预期
